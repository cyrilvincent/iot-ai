TP 8 : HAR Human Activity Recognition
HAR est un dataset disponible ici : https://archive.ics.uci.edu/dataset/240/human+activity+recognition+using+smartphones
Il enregistre les activités de votre téléphone
Une IA essaye de déterminer votre type d'activité parmis 0 WALKING, 1 WALKING_UPSTAIRS, 2 WALKING_DOWNSTAIRS, 3 SITTING, 4 STANDING, 5 LAYING
Les activités enregistrées sont les données de l'accéléromètres sur 3 axes et du gyroscope sur 3 axes toutes les 10ms (100Hz)
Pour le TP nous allons utiliser uniquement l'accéléromètre avec les données notées x, y, z
Pour déterminer une activité nous avons besoins de 128 mesures soit 1.28s
Nous avons donc 3*128=384 features ce qui est trop pour RF
L'algorithme d'abaissement de dimension est le suivant :
sur x, y, z nous prenons une fenêtre mobile de 128 mesures
Nous allons appliquer les 10 fonctions statistiques suivantes sur x, y et 10: median, q25, q75, q75-q25, min, max, std, energy=0, entropy
Cela nous donne 3*10 features
La série étant temporelle nous appliquons une normalisation L1, nous passons donc de 3*128 à 128 features
Sur L1 nous appliquons les 8 fonctions statistiques suivantes median, q25, q75, q75-q25, min, max, std
Cela nous donne 8 features
Nous faisons la même chose avec une normalisation L2 et obtenons 8 features
Nous avons donc obtenu 3*10+8+8=46 features
Sur la séries x,y,z nous appliquons une dérivée (appelée Jerk f(x+1)=f(x+1)-f(x))
Sur cette nouvelle séries nous appliquons les mêmes calculs et obtenons 46 features supplémentaires
Nous avons donc en tout 46+46=92 features (réduction de 384 à 92)
Les capteurs sont 16 bits donc 1.28s = 92*2=184 octets
L'apprentissage est programmé dans har_trees/har_train.py
Les fonctions de réductions sont programmées har_tree/timebase.py
Les fonction de fenêtrage sont programmées dans har_tree/windower.py
La fonction qui traduit des CSV x,y,z en array de 92 features en 16 bits est programmées dans har_csv_to_npy.py
Le modèle a déjà été entrainé et est disponible dans data/har/uci_har_tree.csv
Copier le modèle sur l'ESP32
Transférer les fichiers data/har/uci_har.testdata.* sur l'ESP32
Le fichier NPZ contient 1 minute d'activité où l'utilisateur passe par les différents stades d'activité
Dans ESP32
Copier lib/zipfile.py npyfile.py et timebased sur ESP32
Exécuter har_run.py => accuracy=81.7%
Les erreurs sont souvent dans la passage d'une activité à l'autre à cause de la taille de la fenêtre de 1.28s
